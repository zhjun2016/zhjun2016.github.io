hexo博客使用

前言

使用hexo博客框架搭配个人博客，优点不需要购买服务器，可以使用github或者gitee(码云)做博客服务器。二者都需要申请仓库，并且仓库地址，仓库路径要和用户名一致，才不会产生二级域名。

格式：

用户名+.github.io

注：

  1. 使用github需要加速器stream+，否则打开很卡；使用gitee pages功能，博客才能被外网访问，需要实名认证；二者搭建流程相似；
  2. github或者gitee做服务器，建议使用ssh key，博客推送服务器时不需输入账号密码；使用https方式推送需要账号密码；

一、工具准备

  1. node.js 安装

​ 官网： https://nodejs.org/zh-cn/ ，根据系统选择合适版本下载，一直下一步安装，新版本自带环境变量；

打开cmd命令，输入“npm -v ” 检查npm版本，输入“node -v” 检查node版本，如果安装成功会显示版本信息；

  1. git安装

​ 阿里淘宝镜像站下载最新的版本，速度较快：https://npm.taobao.org/mirrors/git-for-windows/

​ Git Bash Here 中操作，输入“git version” 显示版本信.

  1. github上创建hexo的仓库

    格式：用户名+.github.io

    eg: git@github.com:username/username.github.io.git

  2. 本机上生成git的ssh key

    右键打开git bash here，进行操作

    $ cd ~/.ssh //查找ssh是否存在

    $ ls

    $ ssh-keygen -t rsa -C “xxxxxxxx@qq.com“ //生成ssh key，一直确认yes

    $ ls //清单

    $ cat id_rsa.pub //查看公钥,添加到github,gitee等

    $ ssh -T git@github.com //验证连接

    保存公钥信息，

  3. github添加ssh key

​ 打开github个人设置，找到GitHub_Settings_keys 页面，新建new SSH Key，添加ssh key的公钥信息；

​ git bash here窗口验证连接 ，

​ ssh -T** git@github.com //验证连接

  1. hexo安装

    建议创建一个博客文件myblog，cmd命令打开下进行安装,

    按照顺序安装：hexo-cli、hexo-deployer-git –save、hexo-image-link –save；

    #hexo脚手架

    $ npm install -g hexo-cli

    #hexo -v来验证是否成功

    #hexo deploy自动部署发布公管局

    $ npm install hexo-deployer-git –save

    #安装渲染工具pug以及stylus, 最新版本hexo，可以不安装

    $ npm install hexo-renderer-pug hexo-renderer-stylus –save

    #安装图片地址插件

    $ npm install hexo-image-link –save

    根目录下的 _config.yml 配置修改：

    post_asset_folder: true

    处理图片连接显示

    #处理标题中文链接，（Hexo的默认文章链接格式是年，月，日，标题这种格式来生成的）

    $ npm install hexo-abbrlink –save

二、hexo使用

  1. 官网

​ hexo官方文档：https://hexo.io/zh-cn/docs/configuration, 了解命令，文档格式；

​ 网站信息配置修改，在/_config.yml配置中，**.yml格式文件，需要遵守k: v书写规范，冒号后面必须有一个空格**， 否则语法报错。

2.博客目录

在博客目录的 source 文件夹中的 _post 文件夹中添加你要写的博客文件

  • _config.yml：俗称站点配置文件，很多与博客网站的格式、内容相关的设置都需要在里面改。位于根目录下。
  • node_modules:存储Hexo插件的文件，可以实现各种扩展功能。一般不需要管。
  • package.json：插件配置
  • scaffolds：模板文件夹，里面的

post.md文件可以设置每一篇博客的模板。具体用起来就知道能干嘛了。

  • source：非常重要。所有的个人文件都在里面！
  • themes：主题文件夹，可以从Hexo主题官网或者网上大神的Github主页下载各种各样美观的主题

建议用Typora！，有破解版

3.修改_config.yml配置信息

​ 设置网站标题、作者、关键字，url，设置语言为中文，设置部署方式

设置网站信息

# Site

title: 网站标题
subtitle: ‘网站子标题’
description: ‘网站描述’
keywords:
author: 作者
language: zh-CN

#设置语言为中文

timezone: ‘Asia/Shanghai’

设置连接地址

url: https://zhjun2016.github.io/

设置主题

theme: butterfly

设置部署方式，以及部署地址

deploy:

​ type: git

​ repo: git@github.com:username/username.github.io.git

​ branch: main

2.常见命令

hexo new “postName” #新建文章

hexo new page “pageName” #新建页面

hexo generate #生成静态页面至public目录

hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）

hexo deploy #部署到GitHub

hexo help # 查看帮助

hexo version #查看Hexo的版本

hexo clean #清理缓存

缩写：

hexo n == hexo new

hexo g == hexo generate

hexo s == hexo server

hexo d == hexo deploy

组合命令：

hexo s -g #生成并本地预览

hexo d -g #生成并上传

hexo new “my first blog” #新建文章

3.2 清理缓存

hexo clean

3.3 生成静态文章

​ hexo g

3.4 启动本地服务器

​ hexo s

3.5 打开本地服务器

​ http://localhost:4000/

3.6部署发布博客

​ 确认博客预览显示正常，可以部署到github上

​ hexo d

3.7 访问远程博客

​ https://username.github.io/

​ 注：可使用组合命令 hexo s -g 、hexo d -g、 hexo clean && hexo g && hexo d && hexo s

  • 打开本地服务器

  • 访问远程服务器

    https://username.github.io/ （username:你github的账号名）

四、踩坑经历

1.npm安装插件报错，部分报错不用理会;

2.安装hexo-asset-image的依赖,不要安装，安装导致图片不能显示;

npm install https://github.com/CodeFalling/hexo-asset-image –save

3.在markdown文件，网络图片不显示

解决方法 ：只需要在markdown文章开头添加一个<meta name="referrer" content="no-referrer" />

4.网站标题格式，如果不是 hexo n命令创建的md文章，需要文章顶部添加头部格式;

5.使用typora免费版本，并选择PicGo作为图床，需处理下偏好设置，否则网络图片不显示；

  1. 浏览器图片不显示，按F12查看下图片地址是否正确，打开图片地址是否正常访问，能访问，但是博客图片不显示，就是配置文件问题，拷贝下原始的配置文件 -config.yml

    然后就是hexo clean && hexo g && hexo d上传博客，正常显示；

  2. github可以添加多个ssh key，每个ssh key对应一个账号的电脑；

  3. 重点介绍source文件夹。新建的博客中，source文件夹下默认只有一个子文件夹——_posts。我们写的博客都放在这个子文件夹里面。我们还可以在source里面新建各种子文件夹满足自己的个性化需求，先搞主线任务上，然后再来搞细节。
